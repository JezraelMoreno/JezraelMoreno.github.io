Flujo de Trabajo con GitHub

A continuación, se describe el flujo de trabajo recomendado al utilizar GitHub con Git.

Pasos para trabajar con GitHub

  1. Crear una cuenta en GitHub:
    • Regístrate en GitHub.
    • Configura tu perfil y preferencias.
  2. Configurar SSH en tu equipo:
    • Abre la terminal y genera una clave SSH con: 
      ssh-keygen -t rsa -b 4096 -C "tu correo electronico aquí"
    • Agrega la clave pública a tu cuenta de GitHub en Settings > SSH and GPG keys.
    • Verifica la conexión con: 
      ssh -T git@github.com
  3. Crear un repositorio en GitHub:
    • Ve a Repositories y haz clic en New.
    • Ponle un nombre y selecciona la licencia MIT.
  4. Clonar el repositorio en tu computadora:
    • En la terminal, usa el siguiente comando (reemplaza la URL con la de tu repositorio): 
      git clone [URL_DEL_REPOSITORIO]
    • Accede a la carpeta del repositorio clonado: 
      cd [Nombre del repositorio]
  5. Hacer cambios en el proyecto:
    • Edita los archivos con un editor de código.
    • Verifica los cambios realizados con: 
      git status
  6. Añadir y confirmar los cambios:
    • Agregar todos los archivos modificados: 
      git add .
    • Confirmar los cambios con un mensaje descriptivo: 
      git commit -m "Descripción de los cambios"
  7. Subir los cambios al repositorio remoto:
    • Enviar los cambios a GitHub: 
      git push origin main
  8. Actualizar el repositorio local con cambios remotos:
    • Si hay cambios en GitHub hechos por otros colaboradores, usa: 
      git pull origin main
